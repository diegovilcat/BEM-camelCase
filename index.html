Metodologías de nombrado: BEM y camelCase

Esta página explica qué son BEM y camelCase, sus usos y diferencias.

¿Qué es BEM?

BEM significa Block Element Modifier. Es una metodología para nombrar clases en CSS de forma organizada.

  1. Organiza mejor el código
  2. Fácil de mantener
  3. Evita conflictos de nombres
  • .boton
  • .boton__icono
  • .boton--grande

¿Qué es camelCase?

camelCase es una forma de escribir nombres sin espacios, usando mayúsculas para separar palabras.

  1. En JavaScript para nombres de variables
  2. En funciones
  3. En propiedades de objetos
  • miVariable
  • nombreCompleto
  • calcularEdad

Comparación entre BEM y camelCase

Característica BEM camelCase
Uso común CSS JavaScript
Formato bloque__elemento--modificador nombreVariable
Legibilidad Alta (estructura clara) Media (palabras unidas)
Separadores Guiones bajos y dobles guiones Mayúsculas en medio de palabras
Recomendado en Proyectos grandes con mucho CSS Funciones y variables en JS
Facilidad para reutilizar código Alta (modificadores reutilizables) Baja (más enfocado a JS)
Evita conflictos de nombres Sí No siempre

Usa BEM para nombrar clases en CSS y camelCase para variables y funciones en JavaScript.
